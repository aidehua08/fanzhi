# 泛知科技首页设计说明：企业 AI 经营舱

## 目标

首页采用“泛知混合版企业 AI 经营舱”方向：吸收 Remix 的粒子地形、光轨和故障视觉，以及 Sazabi 的未来驾驶舱、沉浸入口和系统感，但最终表达必须服务于泛知科技的业务定位。

页面第一眼要传达：泛知科技帮助中小企业老板从 AI 咨询、企业培训走到 Agent 开发、知识库建设和落地陪跑。视觉上要像进入一个未来企业 AI 操作系统，而不是传统科技官网，也不是纯炫技作品集。

## 受众

主要用户是中小企业老板和管理者。他们不一定懂技术，但关心业务增长、效率、管理、客服、销售和团队执行。首页可以很酷，但每个视觉隐喻都要能对应到“企业如何用上 AI”。

## 核心创意

首屏是一个“企业 AI 经营舱”：

- 中心是 AI 诊断核心，类似雷达、能量核、操作系统入口。
- 背景是暗色未来空间，包含粒子网格、光轨、数据流和轻微故障扫描。
- 鼠标移动时，画面整体有视差，像用户正在操控系统。
- 鼠标附近出现扫描光或跟踪点，暗示系统正在识别企业痛点。
- 点击时产生能量波或雷达扩散，暗示启动诊断。
- 指标卡片围绕中心漂浮，显示企业 AI 落地相关指标和场景。

## 视觉语言

颜色：

- 背景：深黑蓝 `#020617` / `#07111F`
- 主发光色：电青 `#5EEAD4`
- 数据蓝：冷蓝 `#38BDF8`
- 警示/行动色：霓虹红 `#FF3B57`，只少量用于高亮
- 文本：冷白 `#E5FAFF`
- 弱文本：蓝灰 `rgba(226, 232, 240, 0.68)`

质感：

- 暗色驾驶舱
- 细网格和粒子
- 半透明 HUD 卡片
- 光轨和扫描线
- 轻微故障字效果，避免过度赛博朋克

不使用：

- 旧 Logo 图片、门头图片、门牌图片
- 大面积紫色渐变
- 传统 SaaS 卡片堆叠首屏
- 纯装饰性 3D 人像

## 动效设计

首屏加载：

1. 黑场进入，背景粒子和光轨先出现。
2. 中心 AI 核心缩放显现。
3. 标题做轻微故障错位后稳定。
4. 指标卡片依次浮入。
5. 按钮最后出现。

鼠标移动：

- 背景粒子场轻微跟随鼠标视差。
- 中心 AI 核心朝鼠标方向偏移 8-16px。
- 指标卡片以不同强度偏移，形成深度层次。
- 鼠标周围出现柔和扫描光晕。

点击反馈：

- 点击页面或主按钮时，从点击点扩散一圈雷达波。
- 中心 AI 核心短暂增强亮度。
- 部分连接线快速流动一次。

滚动过渡：

- 首屏向下滚动时，AI 核心逐渐缩小并淡出。
- 下一段进入“AI 落地，不止培训”的三阶段链路。
- 首页的视觉动能要自然过渡到咨询、培训、落地三步。

## 技术方案

推荐用 React + TypeScript + Tailwind 延续当前项目。

首屏视觉分两层：

1. WebGL / Canvas 层  
   用 Three.js 或 React Three Fiber 实现粒子地形、光轨、核心能量球或扫描网格。

2. DOM/HUD 层  
   用 React + Framer Motion 实现导航、标题、按钮、指标卡、点击波纹和滚动过渡。

优先级：

- 第一版先实现高质量 2.5D / Canvas 效果，确保性能和可控性。
- 如果第一版效果不足，再升级 Three.js 粒子地形。

性能边界：

- 桌面端允许 WebGL 粒子和鼠标视差。
- 移动端降级为 CSS/Canvas 静态轻动效，减少粒子数量。
- 尊重 `prefers-reduced-motion`，关闭大幅动画和持续运动。
- 首屏必须在本地和常见浏览器中流畅运行。

## 组件拆分

建议拆分：

- `HeroCockpit`：首屏总容器。
- `CockpitCanvas`：粒子、光轨、扫描核心。
- `HeroHud`：标题、按钮、指标卡。
- `PointerAura`：鼠标光晕和点击波纹。
- `FloatingMetricCard`：可复用指标卡。
- `usePointerParallax`：统一管理鼠标位置和视差参数。

## 验收标准

桌面端：

- 打开页面第一眼像进入未来企业 AI 系统。
- 鼠标移动有明确跟随感，但不晕、不乱。
- 点击反馈明显。
- 标题和业务定位清楚可读。
- 不出现旧 Logo、门头、门牌图片。

移动端：

- 首屏文字完整可读。
- 动效保留氛围但不影响滑动。
- 不出现元素重叠或按钮溢出。

技术验证：

- `npm run build` 通过。
- `npm run lint` 通过。
- 浏览器无运行错误。
- 桌面和移动截图检查通过。

## 当前决策

已确认采用 C 方案：泛知混合版企业 AI 经营舱。

暂不采用纯 Remix 式粒子地形，因为它偏框架/开发者叙事，容易抽象。

暂不采用纯 Sazabi 式未来城市驾驶舱，因为它制作成本高，且容易让用户以为是游戏或娱乐产品。

最终方向是：保留两者的酷感，但让视觉隐喻落到“企业 AI 诊断、培训、Agent、知识库、落地陪跑”。
